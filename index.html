Game-Playing Framework and Intelligent Game-Playing Agents

My undergraduate thesis in Computer Science. Included is a general purpose games-playing framework complete with 2 games, Go and Hex. Also included are a trivial agent and 4 intelligent agents which can play the games. The trivial agent simply makes random moves on the board, while the four other agents use Monte-Carlo tree search. One uses a vanilla MCTS, while the other three each use either an artificial neural network, genetic algorithm, or neuro-evolved network to bias the MCTS algorithm. Libraries used include JCommander and the Encog machine learning library. Following the creation of the framework, games, and agents, I performed research analyzing the tradeoff between the performance and computational complexity of each agent.

  • Language: Java
  • Date: Fall 2016 - Spring 2017
  • Source Code: Available on Github

Raspberry Pi Surveillence

Using a Raspberry Pi camera and the OpenCV computer vision library, I created a security camera with two different functions. The camera can function as a basic motion detector, and works by tracking changes in the frame over time and signaling that the room is occupied if a certain difference threshold is met between frames. Second, a histogram-of-gradients (HOG) has been trained to recognize an actual human figure, and each frame of the camera's recording is ran through the HOG in real-time, signaling the room is occupied whenever a human figure is detected. When a human figure is detected, the frame is saved and uploaded to dropbox.

  • Language: Python
  • Date: Fall 2016
  • Source Code: Available on Github

OpenStack Deployment

A friend and I convinced Allegheny's CS department to lend us five computers to attempt to deploy OpenStack to them and provide infrastructure-as-a-service to the department. We used Ubuntu MAAS to configure the cluster and bootstrap Juju, a service orchestration tool. Using Juju, we then successfully deployed all of the needed OpenStack software across the five machines. After deploying OpenStack, we learned the school's network security settings would not allow connection to the cluster over the school's network; despite this, it was still a fun experience poking into the world of distributed systems.
